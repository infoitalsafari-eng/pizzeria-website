TÂCHE

Étape 1 - Authentification admin

Étape 2 - Brancher Admin.tsx sur Supabase (lecture + écriture)
Remplace useApiService dans Admin.tsx par des appels Supabase pour :

Utilise la clé anon côté client — la sécurité d'écriture doit reposer sur les 
policies RLS déjà en place (utilisateur authentifié = admin), pas sur une clé 
secrète exposée au frontend.

Étape 3 - Migrer le backend api.mjs

un client Supabase utilisant la clé service_role (déjà stockée en secret Replit).

(vérifie en cherchant tous les usages de useApiService dans le projet), propose-moi 
de le supprimer entièrement plutôt que de le migrer — confirme ce point avant d'agir.

Étape 4 - Nettoyage Firebase complet

Étape 5 - Sécurité .env
Le fichier .env est actuellement commité dans git avec des clés sensibles. 
- Ajoute .env à .gitignore (s'il n'y est pas déjà)
